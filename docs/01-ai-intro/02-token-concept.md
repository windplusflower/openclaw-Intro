# Token 概念

> 理解 AI 如何计费

---

## 💰 什么是 Token

**Token** 是 AI 处理文字的基本单位。

### 通俗理解

- 📝 **1 个 Token** ≈ 1.5 个英文单词
- 📝 **1 个 Token** ≈ 0.5-1 个中文字
- 📝 **1000 个 Token** ≈ 750 个英文单词 ≈ 500-1000 个中文字

---

## 🔢 Token 如何计算

### 英文示例

```
输入："Hello, how are you?"

分词结果：
["Hello", ",", " how", " are", " you", "?"]

Token 数：6 个
```

### 中文示例

```
输入："你好，今天天气怎么样？"

分词结果：
["你", "好", "，", "今天", "天气", "怎么", "样", "？"]

Token 数：8 个
```

## 💵 Token 计费方式

### 计费规则

AI 服务按 Token 数量收费：

```
总费用 = 输入 Token 数 × 输入价格 + 输出 Token 数 × 输出价格
```

```
处理 100 万字（约 1M tokens）的费用：

国外模型（GPT-5.4）：约 $17.50（¥125）
国外模型（Claude Opus 4.6）：约 $30（¥215）
国外模型（Gemini 3.1 Pro）：约 $14（¥100）
国内模型（DeepSeek）：约 ¥10-20

💰 差价：5-15 倍！（2026 年价格已大幅下降）
```

### 实际花费示例

```
对话：
用户："今天天气怎么样？" (8 Token)
AI："今天天气晴朗，气温适宜，适合外出活动。" (20 Token)

花费计算（以 DeepSeek 为例）：
输入：8 Token × ¥0.001/1000 = ¥0.000008
输出：20 Token × ¥0.002/1000 = ¥0.00004
总计：¥0.000048（几乎免费）
```

## 📊 Token 消耗对比

### 短对话 vs 长对话

| 对话类型 | 输入 Token | 输出 Token | 总 Token | 预估花费 |
|----------|-----------|-----------|---------|----------|
| **简单问答** | 10 | 20 | 30 | ¥0.001 |
| **中等对话** | 100 | 300 | 400 | ¥0.013 |
| **长文档分析** | 2000 | 1000 | 3000 | ¥0.05 |
| **完整文章** | 50 | 5000 | 5050 | ¥0.15 |

### 省钱技巧

| 技巧 | 说明 | 节省 |
|------|------|------|
| **简洁提问** | 避免冗长的背景描述 | 30-50% |
| **指定长度** | "用 100 字总结" | 50-70% |
| **分批处理** | 长文档分段发送 | 20-30% |
| **复用上下文** | 在同一会话中继续 | 10-20% |

## 🎯 上下文窗口

### 什么是上下文窗口

AI 每次能处理的 Token 数量有限制：

```
上下文窗口 = 输入 Token + 输出 Token 的最大值
```

### 2026 年 3 月主流模型上下文窗口

| 模型 | 上下文窗口 | 相当于 |
|------|-----------|--------|
| **GPT-5.4** | 1.05M | 约 80 万字的小说 |
| **Claude Opus 4.6** | 1M | 约 80 万字的小说 |
| **Claude Sonnet 4.6** | 1M | 约 80 万字的小说 |
| **Gemini 3.1 Pro** | 1M | 约 80 万字的小说 |
| **DeepSeek V3** | 128K（灰度测试 1M） | 约 10 万字的小说 |
| **Qwen2.5-1M** | 1M（100 万 tokens） | 约 80 万字的小说 |
| **豆包 1.5 Pro** | 256K | 约 20 万字的小说 |

### 上下文窗口对比理解

```
📚 一本普通小说：约 10-20 万字
📚 《红楼梦》：约 100 万字
📚 长篇论文：约 1-5 万字
📄 一份合同：约 5000-20000 字
```

### 超出限制会怎样

```
❌ 错误：Token 数量超出限制

解决方法：
1. 缩短输入内容
2. 分批发送
3. 使用支持更大上下文的模型（如 Qwen2.5-1M）
```

1. "你好"

### 示例 2：计算花费

假设价格：输入 ¥0.01/1000 Token，输出 ¥0.03/1000 Token

```
对话：
用户：发送了一篇 1000 字的文章
AI：生成了 500 字的总结

计算总花费。
```

<details>
<summary>点击查看答案</summary>

输入：1000 字 ≈ 1500 Token
输出：500 字 ≈ 750 Token

输入费用：1500 × 0.01/1000 = ¥0.015
输出费用：750 × 0.03/1000 = ¥0.0225
总计：¥0.0375（约 4 分钱）
</details>

## 💡 省钱建议

### ✅ 这样做更省钱

1. **问题要具体**
   ```
   ❌ "帮我写点什么"
   ✅ "写一封 100 字的请假邮件"
   ```

2. **指定格式和长度**
   ```
   ❌ "总结一下这篇文章"
   ✅ "用 3 句话总结，每句不超过 20 字"
   ```

3. **利用上下文**
   ```
   ❌ 每次都重新描述背景
   ✅ 在同一会话中继续对话
   ```

### ❌ 这样做浪费钱

1. **重复发送相同内容**
2. **问题模糊导致 AI 乱猜**
3. **生成超长内容然后不用**

## ⚠️ 重要提醒

### Token 是双向计费

- 📥 **输入 Token**：你发送的内容
- 📤 **输出 Token**：AI 生成的内容
- 💰 **两者都收费**

### 隐藏成本

| 隐藏成本 | 说明 |
|----------|------|
| **系统提示** | 每次对话都有隐藏的 system prompt |
| **会话历史** | 同一会话中的历史对话也计费 |
| **重试** | 每次重试都重新计费 |
