# Token 概念

> 理解 AI 如何计费

---

## 💰 什么是 Token

**Token** 是 AI 处理文字的基本单位。

### 通俗理解

- 📝 **1 个 Token** ≈ 1.5 个英文单词
- 📝 **1 个 Token** ≈ 0.5-1 个中文字
- 📝 **1000 个 Token** ≈ 750 个英文单词 ≈ 500-1000 个中文字

## 🔢 Token 如何计算

### 英文示例

```
输入："Hello, how are you?"

分词结果：
["Hello", ",", " how", " are", " you", "?"]

Token 数：6 个
```

### 中文示例

```
输入："你好，今天天气怎么样？"

分词结果：
["你", "好", "，", "今天", "天气", "怎么", "样", "？"]

Token 数：8 个
```

## 💵 Token 计费方式

### 计费规则

AI 服务按 Token 数量收费：

```
总费用 = 输入 Token 数 × 输入价格 + 输出 Token 数 × 输出价格
```

### ⚠️ 实际 Token 消耗：远不止表面看起来那么简单！

**重要认知纠正**：

很多新手误以为 Token 消耗就是「你说一句，AI 回一句」那么简单。但实际上，**现代 AI 助手（如 OpenClaw）的工作方式要复杂得多**。

#### 错误认知 vs 实际情况

| 场景 | 错误认知 | 实际情况 |
|------|----------|----------|
| **用户说一句话** | 只产生 1 次请求 | 可能产生 **3-10 次** API 请求 |
| **Token 消耗** | 几十到几百 | **十几万到几十万** |
| **费用估算** | 几乎免费 | **可能几毛钱到几块钱** |

#### 为什么实际消耗这么高？

**1. 工具调用机制**

当你让 AI 帮你完成一个任务时，它可能需要：
```
用户："帮我查一下今天的天气"

AI 内部流程：
1. 分析用户意图（第 1 次请求）
2. 决定调用天气查询工具（第 2 次请求）
3. 执行工具调用，获取天气数据（第 3 次请求）
4. 整理数据，生成回复（第 4 次请求）
5. 可能还需要验证结果（第 5 次请求）
```

**2. 上下文累积效应**

```
第 1 轮对话：
- 系统提示词：2000 tokens
- 用户输入：50 tokens
- AI 输出：200 tokens
- 总计：2250 tokens

第 2 轮对话：
- 系统提示词：2000 tokens
- 第 1 轮历史：250 tokens（累积）
- 用户输入：50 tokens
- AI 输出：200 tokens
- 总计：2500 tokens

第 10 轮对话：
- 历史累积可能达到 5000+ tokens
- 每次请求都要带上全部历史！
```

**3. 复杂任务的链式调用**

对于复杂任务（如"帮我分析这个项目的代码"）：
```
AI 可能执行：
1. 列出目录结构（工具调用）
2. 读取关键文件（多次文件读取）
3. 分析代码逻辑（多次推理）
4. 搜索相关文档（网络搜索）
5. 生成总结报告（最终输出）

总 Token 消耗可能达到：50,000 - 200,000 tokens！
```

假设你正在使用 OpenClaw 进行类似本次的对话：


#### 影响 Token 消耗的因素

| 因素 | 说明 | 影响程度 |
|------|------|----------|
| **工具调用次数** | 每次调用都消耗大量 tokens | ⭐⭐⭐⭐⭐ |
| **上下文长度** | 历史对话越长，每次请求消耗越多 | ⭐⭐⭐⭐ |
| **文件操作** | 读取/写入大文件消耗大量 tokens | ⭐⭐⭐⭐ |
| **模型选择** | 不同模型的系统提示词长度不同 | ⭐⭐⭐ |
| **任务复杂度** | 复杂任务需要更多推理步骤 | ⭐⭐⭐⭐⭐ |

#### 如何合理估算成本？

**简单估算公式**：
```
实际成本 = 表面 Token 数 × 10-50 倍 × 单价

示例：
- 表面看起来：1000 tokens
- 实际消耗：10,000 - 50,000 tokens
- 按 ¥2/百万 tokens 计算：¥0.02 - ¥0.1
```

**更准确的估算**：
```
简单问答（无工具调用）：1x - 3x 表面 Token
中等任务（少量工具调用）：5x - 15x 表面 Token
复杂任务（多次工具调用）：20x - 50x 表面 Token
代码分析/文件操作：30x - 100x 表面 Token
```

### 简单的花费示例（仅作参考）

```
对话：
用户："今天天气怎么样？" (8 Token)
AI："今天天气晴朗，气温适宜，适合外出活动。" (20 Token)

⚠️ 注意：这是最简单的场景，实际 AI 助手可能还需要查询天气 API！

总 Token 可能达到：500 - 2000 tokens
实际费用：¥0.001 - ¥0.004
```

## ⚠️ 重要提醒

### Token 是双向计费

- 📥 **输入 Token**：你发送的内容
- 📤 **输出 Token**：AI 生成的内容
- 💰 **两者都收费**

### 隐藏成本（非常重要！）

| 隐藏成本 | 说明 | 影响 |
|----------|------|------|
| **系统提示词** | 每次对话都有隐藏的 system prompt，通常 1000-3000 tokens | 每次请求都包含 |
| **工具调用** | AI 调用工具时需要发送工具描述和参数，消耗大量 tokens | 每次工具调用额外消耗 500-2000 tokens |
| **上下文累积** | 同一会话中的历史对话会累积，每次请求都要带上 | 随着对话进行，消耗指数增长 |
| **文件内容** | 读取文件时，文件内容会作为输入 tokens | 大文件可能一次性消耗数万 tokens |
| **重试机制** | 每次重试都重新计费 | 网络不稳定时费用翻倍 |
| **多轮推理** | 复杂任务可能需要 AI 多次"思考" | 每轮思考都消耗 tokens |

### 💡 成本优化建议

**1. 定期开启新会话**
```
当会话历史过长时，开启新会话可以清除累积的上下文
告诉 AI："开启一个新会话"
```

**2. 避免频繁的文件操作**
```
❌ 多次读取同一个大文件
✅ 一次性读取，在上下文中引用
```

**3. 合理选择模型**
```
简单任务用轻量级模型（如 DeepSeek V3）
复杂任务再用大模型（如 Claude/GPT）
```

**4. 监控 Token 使用**
```
问 AI："帮我看看今天的 Token 消耗情况"
```

## 🔑 API 和 API Key 基础概念

在了解了 Token 计费之后，让我们继续学习如何接入 AI 服务——这需要用到 **API** 和 **API Key**。

### 什么是 API？

**API**（Application Programming Interface，应用程序编程接口）就像是**餐厅的服务员**。

想象你去餐厅吃饭：
- 你不需要亲自去厨房做饭（那太复杂了）
- 你只需要告诉服务员你想吃什么
- 服务员把你的需求传达给厨房
- 厨房做好后，服务员把菜端给你

**AI 的 API 也是一样的道理**：
- 你不需要自己搭建一个 AI 模型（那需要超级计算机和海量数据）
- 你只需要通过 API 发送你的问题
- API 把你的问题传给 AI 公司的服务器
- AI 处理完后，通过 API 把回答返回给你

**生活中的 API 例子**：

| 场景 | 类比 |
|------|------|
| 外卖 App | 你下单 → 外卖平台 API → 餐厅 → 骑手送餐 |
| 天气 App | 你打开 App → 调用气象局 API → 获取天气数据 |
| 地图导航 | 输入目的地 → 调用地图 API → 返回路线 |
| OpenClaw | 你输入问题 → 调用 AI API → 返回 AI 回答 |

### 什么是 API Key？

**API Key** 就像是你的**身份证 + 钥匙 + 密码**的组合。

**用餐厅的例子理解**：
- 你是 VIP 会员，有一张**会员卡**（API Key）
- 这张卡证明你是谁（身份认证）
- 这张卡能打开餐厅的门（访问权限）
- 这张卡记录你消费了多少（计费依据）

**API Key 的作用**：

| 作用 | 说明 | 类比 |
|------|------|------|
| **身份认证** | 证明你是谁 | 身份证 |
| **访问控制** | 确定你能使用哪些功能 | 门禁卡 |
| **计费统计** | 记录你的使用量，用于收费 | 电表/水表 |
| **安全防护** | 防止别人冒充你使用 | 密码 |

### 为什么需要 API Key？

**1. 身份认证 - 证明"你是你"**
```
就像寄快递需要写清楚寄件人和收件人一样，
API Key 让 AI 服务知道"这个请求是谁发的"。
```

**2. 计费统计 - "用了多少，付多少钱"**
```
AI 服务是按使用量收费的（按 Token 数量）。
API Key 就像你的"账号"，记录你用了多少，月底一起结算。
```

**3. 权限控制 - "你能做什么"**
```
不同的 API Key 可能有不同的权限：
- 普通用户：只能使用基础模型
- 付费用户：可以使用高级模型
- 管理员：可以管理其他用户
```

**4. 安全防护 - "防止被盗用"**
```
如果你的 API Key 泄露了，别人就能冒充你使用 AI 服务，
产生的费用都算在你的账上！所以 API Key 要像密码一样保管好。
```

### 🔐 API Key 安全须知

**⚠️ 绝对不要做**：
- ❌ 把 API Key 截图发到网上
- ❌ 把 API Key 写在公开代码里（GitHub 等）
- ❌ 把 API Key 发给陌生人
- ❌ 在公共电脑上保存 API Key

**✅ 正确做法**：
- ✅ 获取 API Key 后，**让 AI 帮你配置**（不要手动编辑配置文件）
- ✅ 定期更换 API Key（如果怀疑泄露）
- ✅ 如果发现异常使用，立即在平台禁用该 Key

### ❓ 常见 Q&A

#### Q1: API Key 泄露了怎么办？

**立即采取以下措施**：
1. **马上登录 AI 服务商平台**（如 OpenAI、阿里云百炼等）
2. **删除/禁用泄露的 API Key**
3. **创建新的 API Key**
4. **告诉你的 AI 助手**："我的 API Key 泄露了，刚换了新的，帮我更新配置"
5. **检查账单**，确认没有异常消费

> 💡 **小提示**：大多数平台都支持创建多个 API Key，建议定期轮换。

#### Q2: 哪里获取 API Key？

| 服务商 | 获取地址 | 大致流程 |
|--------|----------|----------|
| **OpenAI** | platform.openai.com | 注册账号 → 绑定支付方式 → 创建 API Key |
| **Anthropic** | console.anthropic.com | 注册账号 → 验证身份 → 创建 API Key |
| **阿里云百炼** | bailian.console.aliyun.com | 登录阿里云 → 开通百炼服务 → 创建 API-KEY |
| **百度千帆** | cloud.baidu.com | 注册百度智能云 → 开通千帆平台 → 创建应用 |
| **DeepSeek** | platform.deepseek.com | 注册账号 → 充值 → 创建 API Key |

#### Q3: API Key 和账号密码有什么区别？

| 对比项 | 账号密码 | API Key |
|--------|----------|---------|
| **用途** | 登录网站/App | 程序调用服务 |
| **使用方式** | 手动输入 | 自动发送 |
| **格式** | 自己设置 | 系统自动生成 |
| **泄露风险** | 高（可能被盗号） | 极高（直接产生费用） |
| **更换频率** | 偶尔 | 建议定期更换 |

#### Q4: 一个 API Key 可以在多台设备上使用吗？

**可以，但不建议**。

- 技术上：一个 API Key 可以在多台电脑、多个程序同时使用
- 安全上：如果其中一台设备泄露，所有设备都受影响
- 管理上：难以追踪哪台设备产生了哪些费用

**建议做法**：
- 每台设备使用独立的 API Key
- 给每个 Key 起个描述性的名字（如"MacBook-Home"、"Windows-Office"）

#### Q5: API Key 有有效期吗？

**取决于服务商**：
- 大多数 API Key **没有固定有效期**，可以长期使用
- 但服务商可能会**定期要求更新**（如每 6 个月）
- 如果**长期不使用**，部分服务商可能会禁用
- 如果**账号欠费**，API Key 会被立即停用
