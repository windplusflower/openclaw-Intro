# Token 概念

> 理解 AI 如何计费

---

## 💰 什么是 Token

**Token** 是 AI 处理文字的基本单位。

### 通俗理解

- 📝 **1 个 Token** ≈ 1.5 个英文单词
- 📝 **1 个 Token** ≈ 0.5-1 个中文字
- 📝 **1000 个 Token** ≈ 750 个英文单词 ≈ 500-1000 个中文字

---

## 🔢 Token 如何计算

### 英文示例

```
输入："Hello, how are you?"

分词结果：
["Hello", ",", " how", " are", " you", "?"]

Token 数：6 个
```

### 中文示例

```
输入："你好，今天天气怎么样？"

分词结果：
["你", "好", "，", "今天", "天气", "怎么", "样", "？"]

Token 数：8 个
```

## 💵 Token 计费方式

### 计费规则

AI 服务按 Token 数量收费：

```
总费用 = 输入 Token 数 × 输入价格 + 输出 Token 数 × 输出价格
```

### ⚠️ 实际 Token 消耗：远不止表面看起来那么简单！

**重要认知纠正**：

很多新手误以为 Token 消耗就是「你说一句，AI 回一句」那么简单。但实际上，**现代 AI 助手（如 OpenClaw）的工作方式要复杂得多**。

#### 错误认知 vs 实际情况

| 场景 | 错误认知 | 实际情况 |
|------|----------|----------|
| **用户说一句话** | 只产生 1 次请求 | 可能产生 **3-10 次** API 请求 |
| **Token 消耗** | 几十到几百 | **十几万到几十万** |
| **费用估算** | 几乎免费 | **可能几毛钱到几块钱** |

#### 为什么实际消耗这么高？

**1. 工具调用机制**

当你让 AI 帮你完成一个任务时，它可能需要：
```
用户："帮我查一下今天的天气"

AI 内部流程：
1. 分析用户意图（第 1 次请求）
2. 决定调用天气查询工具（第 2 次请求）
3. 执行工具调用，获取天气数据（第 3 次请求）
4. 整理数据，生成回复（第 4 次请求）
5. 可能还需要验证结果（第 5 次请求）
```

**2. 上下文累积效应**

```
第 1 轮对话：
- 系统提示词：2000 tokens
- 用户输入：50 tokens
- AI 输出：200 tokens
- 总计：2250 tokens

第 2 轮对话：
- 系统提示词：2000 tokens
- 第 1 轮历史：250 tokens（累积）
- 用户输入：50 tokens
- AI 输出：200 tokens
- 总计：2500 tokens

第 10 轮对话：
- 历史累积可能达到 5000+ tokens
- 每次请求都要带上全部历史！
```

**3. 复杂任务的链式调用**

对于复杂任务（如"帮我分析这个项目的代码"）：
```
AI 可能执行：
1. 列出目录结构（工具调用）
2. 读取关键文件（多次文件读取）
3. 分析代码逻辑（多次推理）
4. 搜索相关文档（网络搜索）
5. 生成总结报告（最终输出）

总 Token 消耗可能达到：50,000 - 200,000 tokens！
```

#### 影响 Token 消耗的因素

| 因素 | 说明 | 影响程度 |
|------|------|----------|
| **工具调用次数** | 每次调用都消耗大量 tokens | ⭐⭐⭐⭐⭐ |
| **上下文长度** | 历史对话越长，每次请求消耗越多 | ⭐⭐⭐⭐ |
| **文件操作** | 读取/写入大文件消耗大量 tokens | ⭐⭐⭐⭐ |
| **模型选择** | 不同模型的系统提示词长度不同 | ⭐⭐⭐ |
| **任务复杂度** | 复杂任务需要更多推理步骤 | ⭐⭐⭐⭐⭐ |

#### 如何合理估算成本？

**简单估算公式**：
```
实际成本 = 表面 Token 数 × 10-50 倍 × 单价

示例：
- 表面看起来：1000 tokens
- 实际消耗：10,000 - 50,000 tokens
- 按 ¥2/百万 tokens 计算：¥0.02 - ¥0.1
```

**更准确的估算**：
```
简单问答（无工具调用）：1x - 3x 表面 Token
中等任务（少量工具调用）：5x - 15x 表面 Token
复杂任务（多次工具调用）：20x - 50x 表面 Token
代码分析/文件操作：30x - 100x 表面 Token
```

### 简单的花费示例（仅作参考）

```
对话：
用户："今天天气怎么样？" (8 Token)
AI："今天天气晴朗，气温适宜，适合外出活动。" (20 Token)

⚠️ 注意：这是最简单的场景，实际 AI 助手可能还需要查询天气 API！

花费计算（以 DeepSeek 为例，仅单次请求）：
输入：8 Token × ¥0.001/1000 = ¥0.000008
输出：20 Token × ¥0.002/1000 = ¥0.00004
总计：¥0.000048（几乎免费）

但如果需要查询天气 API：
总 Token 可能达到：500 - 2000 tokens
实际费用：¥0.001 - ¥0.004
```

## 🎯 上下文窗口

### 什么是上下文窗口

AI 每次能处理的 Token 数量有限制：

```
上下文窗口 = 输入 Token + 输出 Token 的最大值
```

## ⚠️ 重要提醒

### Token 是双向计费

- 📥 **输入 Token**：你发送的内容
- 📤 **输出 Token**：AI 生成的内容
- 💰 **两者都收费**

### 隐藏成本（非常重要！）

| 隐藏成本 | 说明 | 影响 |
|----------|------|------|
| **系统提示词** | 每次对话都有隐藏的 system prompt，通常 1000-3000 tokens | 每次请求都包含 |
| **工具调用** | AI 调用工具时需要发送工具描述和参数，消耗大量 tokens | 每次工具调用额外消耗 500-2000 tokens |
| **上下文累积** | 同一会话中的历史对话会累积，每次请求都要带上 | 随着对话进行，消耗指数增长 |
| **文件内容** | 读取文件时，文件内容会作为输入 tokens | 大文件可能一次性消耗数万 tokens |
| **重试机制** | 每次重试都重新计费 | 网络不稳定时费用翻倍 |
| **多轮推理** | 复杂任务可能需要 AI 多次"思考" | 每轮思考都消耗 tokens |

### 💡 成本优化建议

**1. 定期开启新会话**
```
当会话历史过长时，开启新会话可以清除累积的上下文
节省方法：/session new
```

**2. 避免频繁的文件操作**
```
❌ 多次读取同一个大文件
✅ 一次性读取，在上下文中引用
```

**3. 合理选择模型**
```
简单任务用轻量级模型（如 DeepSeek V3）
复杂任务再用大模型（如 Claude/GPT）
```

**4. 监控 Token 使用**
```
使用 openclaw 的 token 统计功能，了解实际消耗
```
