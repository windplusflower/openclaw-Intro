# OpenClaw 是什么

> 理解 OpenClaw 的功能和用途

---

## 🦞 什么是 OpenClaw

**OpenClaw** 是一个开源的 AI 编程助手框架。

### 通俗理解

| 工具 | 功能 | 比喻 |
|------|------|------|
| **网页版 AI** | 聊天对话 | 在餐厅点菜 |
| **OpenClaw** | 自动化 AI 工作流 | 请个私人厨师 |

---

## 🆚 OpenClaw vs 网页版 AI

### 对比表

| 功能 | 网页版 AI | OpenClaw |
|------|----------|----------|
| **对话方式** | 浏览器聊天 | 终端/多渠道 |
| **自动化** | ❌ 手动操作 | ✅ 定时任务、自动回复 |
| **多渠道** | ❌ 仅网页 | ✅ Discord、飞书、钉钉 |
| **技能系统** | ❌ 固定功能 | ✅ 可扩展技能 |
| **会话管理** | ❌ 有限 | ✅ 无限会话 |
| **价格** | 💰 按次付费 | 💰 只需 API Key |

### 使用场景对比

```
场景 1：简单问答
├── 网页版 AI ✅ 适合
└── OpenClaw ⚠️ 大材小用

场景 2：自动回复消息
├── 网页版 AI ❌ 做不到
└── OpenClaw ✅ 适合

场景 3：定时发送日报
├── 网页版 AI ❌ 做不到
└── OpenClaw ✅ 适合

场景 4：多渠道统一管理
├── 网页版 AI ❌ 做不到
└── OpenClaw ✅ 适合
```

## 🎯 OpenClaw 能做什么

### 核心功能

#### 1. 多渠道支持

```
OpenClaw
├── Discord 机器人
├── 飞书机器人
├── 钉钉机器人
├── 终端界面
```

```yaml

```

## 🏗️ OpenClaw 架构

### 核心组件

```
OpenClaw
├── Gateway（网关）
│   └── 核心服务，管理所有功能
├── Session（会话）
│   └── 独立的 AI 对话上下文
├── Channel（渠道）
│   └── Discord、飞书、钉钉等
├── Skill（技能）
│   └── 可扩展的功能模块
└── Agent（代理）
    └── AI 模型连接
```

### 工作流程

```
用户消息
    ↓
Channel（渠道接收）
    ↓
Gateway（网关处理）
    ↓
Session（会话上下文）
    ↓
Skill（技能处理）
    ↓
Agent（AI 生成回复）
    ↓
Channel（渠道发送）
    ↓
用户收到回复
```

## 💰 费用和定价

### OpenClaw 本身

- ✅ **完全免费** - 开源软件
- ✅ **自行部署** - 在自己电脑上运行
- ✅ **无隐藏费用** - 没有订阅费

### API 费用

OpenClaw 需要配置 AI 服务的 API Key：

| 服务商 | 价格范围 | 适合场景 |
|--------|---------|---------|
| **OpenAI** | $0.01-0.10/1000 Token | 通用场景 |
| **Anthropic** | $0.01-0.30/1000 Token | 长文档处理 |
| **百度文心** | ¥0.01-0.10/1000 Token | 中文场景 |
| **阿里通义** | ¥0.01-0.10/1000 Token | 中文场景 |

### 费用估算


## ⚠️ 注意事项

### 技术要求

| 要求 | 说明 |
|------|------|
| **电脑** | macOS 或 Windows |
| **网络** | 能访问 AI 服务 API |
| **基础** | 会用终端/命令行 |
| **时间** | 2-3 小时学习时间 |

### 不适合人群

| 人群 | 原因 |
|------|------|
| **完全零基础** | 需要先学终端基础 |
| **不想配置** | 需要配置 API Key |
| **只要聊天** | 网页版 AI 更简单 |
