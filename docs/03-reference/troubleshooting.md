# 问题排查指南

在使用 OpenClaw 过程中，遇到问题时不要慌张。根据 OpenClaw 是否能正常运行，有两种排查方式：

---

## ⚠️ 重要说明

**让 AI 排查的前提：OpenClaw 已经能正常运行**

- ✅ **安装完成后** - OpenClaw 能启动 → 可以让 AI 帮忙排查
- ❌ **安装失败时** - OpenClaw 起不来 → 需要人工排查（见下方"安装阶段问题"）

---

## 🤖 运行阶段：让 AI 帮你排查（推荐）

**前提条件：** OpenClaw 已经安装完成并能启动

**遇到任何问题，第一反应应该是告诉 AI：**

```
你：我的天气报告今天没有发送，帮我检查一下
AI：好的，我来帮你排查。让我先查看日志...
```

**AI 会帮你：**
- ✅ 自动查看日志
- ✅ 检查配置是否正确
- ✅ 测试各个组件连接
- ✅ 诊断问题并提供解决方案
- ✅ 直接修复配置（如果可以）

**为什么让 AI 排查更好？**
- 🚀 **更快**：AI 知道该看哪里
- 🎯 **更准**：AI 了解常见问题
- 💡 **更简单**：不需要记住复杂命令
- 🔧 **自动修复**：AI 可以直接修复配置

---

## 🔧 安装阶段：人工排查（OpenClaw 无法运行时）

**适用场景：** OpenClaw 还没安装完成，或者安装后无法启动

此时 AI 无法运行，需要人工排查或寻求社区帮助。

### 步骤 1：查看错误信息

仔细阅读终端显示的错误信息，通常会提示问题所在：

```
❌ 常见错误示例：
- "npm: command not found" → Node.js 未安装
- "Permission denied" → 权限不足
- "Network timeout" → 网络问题
- "Unsupported Node.js version" → Node.js 版本不对
```

### 步骤 2：根据错误类型处理

| 错误类型 | 可能原因 | 解决方法 |
|----------|----------|----------|
| **命令找不到** | Node.js/npm 未安装 | 先安装 Node.js（见安装指南） |
| **权限不足** | 需要管理员权限 | 使用 sudo（Mac/Linux）或管理员身份运行（Windows） |
| **网络超时** | 无法访问 GitHub/npm | 检查网络，或配置代理（见 FAQ） |
| **版本不兼容** | Node.js 版本不对 | 安装 Node.js v18 或更高版本 |

### 步骤 3：寻求人工帮助

如果以上方法无法解决：

1. **查看官方文档**
   - 安装指南：[macOS](../02-openclaw-guide/02-install-macos.md) / [Windows](../02-openclaw-guide/03-install-windows.md)
   - 常见问题：[FAQ](./faq.md)

2. **搜索 GitHub Issues**
   - 访问：https://github.com/openclaw/openclaw/issues
   - 搜索类似问题的解决方案

3. **在社区提问**
   - Discord 社区
   - 微信群/QQ 群
   - 提供错误信息和系统环境

4. **提交 Issue**
   - 在 GitHub 仓库提交新 Issue
   - 附上完整的错误日志
   - 说明你的系统环境和操作步骤

## 📋 运行阶段：常见问题及解决

**前提：** OpenClaw 已安装并能启动

### 问题 1：配置问题

**让 AI 帮你：**
> "帮我检查一下配置是否正确"

AI 会验证 YAML 格式、检查配置项、修复错误。

**常见错误：**
- 缩进不正确
- 缺少冒号或引号
- 使用了制表符而非空格

---

### 问题 2：API Key 无效

**让 AI 帮你：**
> "API 连接失败了，帮我检查 Key 是否正确"

AI 会：
- 检查 Key 是否填写正确
- 测试 API 连接
- 帮你重新获取 Key（如果需要）

---

## 📊 查看日志（了解即可）

如果你想了解日志信息，可以让 AI 帮你查看：

**告诉 AI：**
> "帮我看看最近的错误日志"

**日志级别说明：**
- `ERROR` - 严重错误，需要立即处理
- `WARN` - 警告信息，可能影响功能
- `INFO` - 一般运行信息
- `DEBUG` - 调试信息，用于深入排查

---

## 🆘 寻求进一步帮助

如果 AI 也无法解决问题：

**让 AI 帮你：**
> "这个问题好像比较复杂，你帮我整理一下信息，我要去社区提问"

AI 会：
1. 整理错误日志和配置信息（脱敏后）
2. 建议查看官方文档和常见问题
3. 帮你在社区或 Issue 中提问
4. 提供复现步骤和环境信息

遇到问题时，按顺序尝试：

## 💡 预防胜于治疗

**良好的使用习惯可以减少问题：**

1. **定期更新**
   - 让 AI 帮你检查更新："帮我看看有没有新版本"
   
2. **备份配置**
   - 让 AI 帮你备份："帮我备份当前配置"
   
3. **监控状态**
   - 让 AI 定期检查："帮我检查一下系统状态"
   
4. **及时清理**
   - 让 AI 清理缓存："帮我清理一下缓存"
